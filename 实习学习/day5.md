1. 在go的go.mod文件中，导入的通常是其它包的module名，而不是精确到其它包的最深目录下的文件或包。module 名是 Go 语言模块系统的核心概念之一。它用于唯一标识一个模块，在 Go 的依赖管理中扮演着关键角色。例如，一个模块可能名为github.com/user/myproject，当你在go.mod文件中导入这个 module，就表示你要使用该 module下包。在具体的代码中就可以利用`module名+相对路径`的方式来引用具体的包，比如：
    ```go
    module trpc.group/trpc-go/trpc-go
    // 那么其他项目需要导入并使用该模块时，其他项目的go.mod文件中只需添加对该模块的导入，如果该module下面有多个包，如client和server，那么就要按照：
    import (
            "trpc.group/trpc-go/trpc-go/client"
            "trpc.group/trpc-go/trpc-go/server"
        )
    ```
2. go.mod 文件的 replace 指令用于在构建过程中替换一个模块的导入路径或版本，比如：如果你正在开发一个模块 github.com/user/myproject，并且希望使用本地目录 ../myproject 中的代码，可以在项目的go.mod中添加:
   ```go
   replace github.com/user/myproject => ../myproject
   ```
3. []*Element(Element是一个结构体)表示一个切片，而不是一个指针，它的每一个元素都是一个指向Element类型实例的指针，类似数组指针
4. 实习过程中一个完整的工程中各个平台的交互应用：
   * 首先，当前代码作为服务端被调用的服务时会注册到服务器上去，然后部署到gdp上，以便其它服务作为客户端时会调用的后端服务。作为服务端的服务和作为客户端调用的后端服务都是在yaml中配置的
   * 其次，不管是作为服务端服务（即被其它客户端调用的服务）还是作为客户端要去调用其它服务，那么这个服务都需要发现，因此这些服务部署到gdp之后都会注册到北极星上，以便于其它服务利用target地址(polaris://服务名)发现(注意：所有服务都是注册到北极星上的，然后其它服务调用它的时候通过北极星地址调用即可)
   * 其次，工程中某些配置信息可能会写到七彩石上去(这些配置信息后续可能会被更改)，七彩石的配置信息也在yaml中，后续在七彩石上直接改这个配置就可以直接在工程中同步更改，而不需要去重启服务，只要在工程中watch这个配置就行。监听这个七彩石的配置是否更改，可以用一个协程单独去处理
   * 如果要从开发环境直接访问idc环境，那么就要给这个服务配置网关，也是在gdp中配置，这个网关其实就是一个网关服务，这个服务也会注册到北极星上，然后用这个北极星上的地址去访问这个网关服务
5. 在 Go 中，`filepath.Join("a", key)`和`filepath.Join("a/", key)`(a是一个文件夹名)的效果是等价的，最终都会生成一个干净、无多余斜杠的路径

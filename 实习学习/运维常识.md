1. 一个服务器分给多个人用:
   * 虚拟化技术：虚拟化技术是在物理服务器上模拟出多个虚拟机，每个虚拟机都有自己独立的操作系统、硬件资源（如虚拟 CPU、虚拟内存、虚拟硬盘等），就像一个大房间里搭建了许多小隔间
   * 容器化技术：容器化技术（如 Docker）是在操作系统层面实现的虚拟化。它不像虚拟机那样拥有一整套独立的操作系统，而是共享宿主机的操作系统内核。容器将应用程序及其依赖打包在一起，形成一个独立的运行环境。就像在一个房间（宿主机）里划分出不同区域（容器），每个区域都有自己独立的物品。容器资源利用率高，比虚拟机启动速度更快。可以将应用程序及其环境封装成容器镜像，在不同的服务器上快速部署。如果要将某个应用从一个服务器迁移到另一个服务器，只需将容器镜像复制过去并启动容器即可
   * 基于账号隔离：在服务器操作系统中为每个用户创建独立的用户账号，并根据需要分配不同的权限。每个用户通过自己的账号登录服务器，只能访问和操作自己有权限的文件、目录和应用程序
2. 接入机用户之间的隔离:
   * 基于用户账号的隔离：在接入机上为每个用户分配独立的账号，通过操作系统的权限管理机制实现操作隔离。用户只能访问和操作自己有权限的文件、目录和应用程序，一个用户对于别人的操作是不知道的
   * 基于虚拟化技术的隔离：通过虚拟化技术，可以将接入机虚拟成多个独立的虚拟机。每个虚拟机为一个用户提供便捷独立的运行环境，不同用户的操作在各自的虚拟机中进行，互不影响（员工A和员工B都通过各自的用户账号访问Windows接入机。员工A下载了一个文件到自己的用户文件夹C:\Users\UserA中，员工B无法访问该文件夹，因此无法知道员工A下载了什么文件）
   * 基于容器化技术的隔离：容器化技术通过容器引擎（如Docker）将应用程序及其依赖打包成独立的容器。不同用户的容器在隔离的环境中运行，共享接入机的操作系统内核，但彼此之间不会互相干扰
3. 容器镜像是一种包含软件及其所有依赖项的轻量级、可执行的独立软件包。容器镜像包含运行特定软件所需的所有内容，包括操作系统文件、应用程序代码、库文件、配置文件等。这些内容在镜像构建过程中被预先配置和打包，确保镜像在任何支持容器技术的环境中都能正确运行
4. 构建一个docker容器最常用的方法就是基于docker容器镜像（可以直接拉取别人推送到Docker Hub上的容器镜像，比如tensorflow镜像等）
5. 生成docker容器镜像是基于Dockerfile文件中的指令执行的，即docker会按照Dockerfile中的指令逐步执行，最终生成一个镜像，如：
   ```dockerfile
   # 使用官方Python镜像作为基础镜像
    FROM python:3.9-slim
    # 将当前目录下的文件复制到容器的/app目录下
    COPY . /app
    # 设置工作目录为/app
    WORKDIR /app
    # 运行app.py文件
    CMD ["python", "app.py"]
    ```
    构建镜像
    ```bash
    docker build -t hello-docker-image .
    ```
    构建过程中，Docker 会按照Dockerfile中的指令逐步执行，最终生成一个名为hello-docker-image的镜像
6. Docker Hub 是一个由 Docker 公司提供的云服务平台，主要用于 Docker 镜像的托管和服务发现等。开发者可以将自己构建的 Docker 镜像上传到 Docker Hub，方便在不同的机器和环境中快速拉取和使用。例如，一个开发团队可以在 Docker Hub 上创建自己的仓库，将应用程序的各个版本镜像上传，团队成员就可以很方便地从这个仓库获取最新的镜像来运行应用程序
7. 分布式系统中的节点指的是每一个独立的计算机或计算单元（计算节点、存储节点、服务节点等）
8. 单个PC机是可以进行容器化的
9. 服务器和PC机的不同:
    * 处理器：服务器通常配备性能强大、稳定性高的多核处理器，如英特尔至强（Xeon）系列或AMD EPYC系列，而PC机一般搭载面向个人用户的处理器，如英特尔酷睿（Core）系列或AMD锐龙（Ryzen）系列
    * 内存：服务器支持大容量、高性能的 ECC（错误纠正码）内存，能自动检测和纠正内存错误，保障数据完整性和系统稳定性。内存容量通常可扩展至数百 GB 甚至数 TB，满足大量数据的存储和处理需求。PC机内存容量一般为几 GB 到几十 GB，不支持 ECC 内存
    * 存储：服务器采用 RAID（独立磁盘冗余阵列）技术，将多个硬盘组合成一个逻辑单元，提供数据冗余和高性能存储。PC机通常使用单个硬盘或简单的磁盘阵列，数据安全和存储性能依赖硬盘本身质量
    * 等等
